# 📝 [My Python Note] 핵심 개념 및 오답 노트 총정리

## 1. 🚨 파이썬의 제1 문법: 들여쓰기 (Indentation)

파이썬은 공백(들여쓰기)이 곧 코드의 구역을 나누는 **절대적인 명령 규칙**입니다. 눈에 보이는 줄맞춤이 전부가 아닙니다.

* **계단식 자식 구조:** `while:`, `for:`, `if:` 처럼 끝에 땡땡(`:`)이 붙으면, 그 다음 줄은 무조건 **키보드 `Tab` 키를 1번** 눌러 안쪽으로 들여 써야 해당 조건에 포함된 자식 코드가 됩니다.
* **들여쓰기 위치에 따른 결과 변화 (`while`문 예시):**
* **반복 출력:** `print`가 `while` 안쪽에 줄이 맞춰져 있으면 바퀴가 돌 때마다 매번 출력합니다. (결과: `1, 3, 5, 7, 9`)
* **최종 출력:** `print`가 `while` 완전히 바깥(맨 왼쪽 벽)에 붙어 있으면 반복문이 다 끝나고 최종 결과만 딱 한 번 출력합니다. (결과: `10`)


* **애매한 공백 금지:** 스페이스바 1~2칸 등의 애매한 정렬은 `IndentationError`를 발생시킵니다. 반드시 `Tab` 키로 정렬하는 습관을 들이세요.

## 2. 💻 루프(반복문)와 제어문의 핵심 원리

### ① `for` 문과 반복 변수 (`i`, `j`)

* `for` 문은 정해진 횟수만큼 코드를 반복하는 문법입니다.
* 반복 변수(`i`, `j`)는 `range()`가 만드는 숫자를 차례대로 배달받는 바구니입니다.
* **핵심:** 반복 변수는 실제 계산이나 출력에 **사용할 수도 있고**, 단순히 횟수만 채우는 **보조 역할**로만 쓸 수도 있습니다.

### ② `range(시작, 끝, 증감)` 완벽 이해

* `range(1, 6)` $\rightarrow$ `1, 2, 3, 4, 5` (**끝 숫자는 포함되지 않음**)
* `range(5, 0, -1)` $\rightarrow$ `5, 4, 3, 2, 1` (세 번째 숫자는 거꾸로 작아지게 만드는 **증감값**)

### ③ 누적합 (`hap = hap + i`)

* `hap = hap + i` 구조는 "기존 스케치북에 있던 값에 새로운 값 `i`를 더해서 덮어쓰기"를 하는 누적 방식입니다. (시작할 때 반드시 `hap = 0`으로 초기화 필요)

### ④ `if` 문의 본질

* `if` 문은 스스로 반복하는 능력이 없습니다. `for`나 `while`이 숫자를 쪼르륵 던져주면, 그중에서 **조건에 맞는 데이터만 골라내는 '필터(거름망)' 역할**을 합니다.

## 3. 📐 알고리즘적 사고: 다이아몬드(별 찍기) 문제 해결법

코드를 무작정 타이핑하지 말고, **결과 $\rightarrow$ 규칙 찾기 $\rightarrow$ 수식화** 순서로 접근합니다.

1. **화면 제어 규칙:** `print("*")`는 출력 후 자동으로 줄바꿈이 되지만, `print("*", end="")`는 **줄바꿈을 막고 옆으로 이어 붙입니다.** 한 층을 다 그리면 `print()`로 줄바꿈 도장을 찍습니다.
2. **쪼개서 생각하기:** 다이아몬드($\black$) 모양은 **위쪽 삼각형**과 **뒤집힌 아래쪽 삼각형** 2개를 이어 붙인 구조입니다.
3. **수식 도출 (표 만들기):**
* **공백(투명 요정) 규칙:** `number - i` (층이 올라갈수록 공백이 줄어듦)
* **별("*") 요정 규칙:** `2 * i - 1` (홀수 개수로 양옆으로 늘어남)

## 4. 🎁 마술 상자(함수)와 재료 주머니(컨테이너)

함수(`def`)를 만들고 값을 넘겨줄 때, 어떤 주머니(컨테이너)를 쓰느냐에 따라 성격이 바뀝니다.

| 함수 형태 | 컨테이너 종류 | 설명 | 예시 |
| --- | --- | --- | --- |
| `def abc(a, b):` | **기본 변수** | 정해진 개수만큼 딱 들어가는 칸막이 상자 | `abc(5, 3)` |
| `def abc2(*args):` | **튜플 (Tuple)** | 별 1개. 몇 개를 던져주든 다 받아먹는 **무한대 보따리** | `abc2(1, 2, 3, 4)` |
| `def abc3(**kwargs):` | **딕셔너리 (Dict)** | 별 2개. **"이름표=값"** 형태로 매칭해서 받는 상자 | `abc3(name="Alice", age=30)` |

* **match-case 문:** `match operator:` 구조는 들어온 값이 어떤 `case` 문에 해당되는지 확인하고 딱 매칭되는 블록 하나만 정직하게 실행합니다.

## 5. 🛠️ 환경 관리 및 정직한 오타 조심 (실전 팁)

### ① 주피터 노트북 무한 대기 `[*]` 현상

* `input()` 함수가 실행되면 컴퓨터는 사용자가 값을 치고 엔터를 누를 때까지 숨을 참고 대기합니다.
* VS Code 주피터 환경에서는 입력창이 화면 맨 위 중앙(상단 바)에 숨어서 깜빡거리니 반드시 위쪽을 확인하세요.
* 멈추고 싶을 때는 상단의 네모 정지 버튼($\blacksquare$)을 누르거나 키보드 `I, I`를 누릅니다.
* 자동완성 보조창이 잠들었을 때는 `Ctrl + Space`를 누르거나 [Restart Kernel]을 눌러 엔진을 청소합니다.

### ② 시스템 입력 값 `sys.argv`

* `python test.py hello world` 처럼 터미널에서 파일을 실행할 때 뒤에 붙여주는 옵션 값들을 리스트 형태로 가져옵니다.
* `sys.argv[0]`은 언제나 실행한 **파일 이름**이 되고, `[1]`부터 진짜 입력값이 들어옵니다.

### ③ 컴퓨터의 정직함 (쉼표 하나가 가르는 에러)

* `open("새파일.txt" 'w')` 처럼 인자 사이에 **쉼표(`,`)를 빼먹으면**, 파이썬은 문자열을 하나로 합쳐서 `새파일.txtw`라는 엉뚱한 파일을 찾다가 `FileNotFoundError`를 냅니다. 인자 구분은 무조건 쉼표(`,`)입니다.
